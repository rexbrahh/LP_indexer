## What this file is

`.golangci.yml` configures `golangci-lint` for this repository.

## Why this file exists

The repo uses `golangci-lint` to enforce:

- correctness checks (`govet`, `staticcheck`, `errcheck`)
- simplifications and unused code detection (`gosimple`, `unused`)
- formatting/import hygiene (`gofmt`, `goimports`)

Keeping this in a repo-level config makes CI and local lint runs consistent.
